# 课程实践项目——车辆维修管理系统（MySQL+SpringBoot+Vue+Java技术栈）

## 一、概述

车辆维修管理系统是专为车辆维修业务设计的综合性数据库应用系统，旨在提高管理效率、降低运营成本，提供优质服务。系统涵盖车辆信息管理、客户信息管理、维修工单管理、库存管理、财务管理等模块，实现数据实时共享与高效流转。

## 二、系统功能需求

### 1. 用户功能

- 注册/登录系统
- 提交车辆报修信息
- 查询账户、车辆、维修记录、工单信息
- 反馈维修进度与结果（催单、评分）
- 查看历史维修记录
  
  ### 2. 维修人员功能
  
- 登录系统
- 查看账户信息（工种、时薪）
- 接收/拒绝工单（拒绝后自动重新分配）
- 记录维修材料消耗及价格
- 更新维修进度与结果
- 查询历史工时费收入
  
  ### 3. 系统管理员功能
  
- 管理用户、维修人员、车辆、工单信息
- 维护数据一致性与完整性
- 监控数据运行与统计结果
  
  ## 三、系统工作流程
  

1. 用户注册登录，提交维修申请；
2. 系统生成工单，自动分配给维修人员；
3. 维修人员接单，记录材料消耗与进度，提交结果；
4. 工单完成，系统计算费用（工时费+材料费）；
5. 用户查询记录并反馈；
6. 管理员监控维护流程。
  
  ## 四、查询与数据分析需求
  
  ### 1. 基础查询（按角色）
  
  | 角色  | 查询需求 |
  | --- | --- |
  | 用户  | 账户/车辆信息、工单信息、历史维修记录 |
  | 维修人员 | 账户信息（工种/时薪）、当前/历史工单、累计工时费收入 |
  | 管理员 | 全量用户/维修人员/车辆/工单信息、历史记录与工时费发放记录 |
  
  ### 2. 进阶统计与分析
  

- **数据统计**：车型维修次数/费用、故障类型统计、季度成本分析、负面反馈筛选、工种任务占比、未完成工单统计
- **数据维护**：工单状态变更触发费用汇总、用户信息变更同步历史记录、工单删除级联操作
- **事务控制**：批量工单分配事务、月度工时费结算批量写入
- **数据回滚**：操作日志记录，支持问题回溯
  
  ## 五、系统展示与实现说明
  
- **技术栈**：MySQL（数据库）、Spring Boot（后端）、Vue（前端）、Java（开发语言）
- **界面形式**：Web界面（Vue开发），禁止直接操作数据库
- **测试要求**：提供模拟数据，确保模块功能可运行
  
  ## 六、提交内容
  

1. ER图
2. 数据库表结构（含索引）
3. 核心SQL语句说明
4. 存储过程/触发器说明（如有）
5. 系统源代码（后端Spring Boot，前端Vue）
6. 可执行程序包与运行文档
  
  ## 双人小组分工方案（MySQL+SpringBoot+Vue+Java）
  
  ### **分工原则**
  
  前后端分离开发，一人负责后端（Spring Boot+MySQL），一人负责前端（Vue），共同协作数据库设计与联调。
  
  ### **成员A：后端开发（Spring Boot+MySQL）**
  
  #### 1. 数据库设计（核心协作任务）
  

- 设计ER图（用户、维修人员、车辆、工单、材料、财务等实体关系）
- 定义表结构（含字段类型、外键约束、索引优化，如工单状态索引、车型统计索引）
- 实现存储过程与触发器（如工单完成自动计算费用、级联删除逻辑）
- 编写模拟测试数据（初始化用户、车辆、工单数据）
  
  #### 2. 后端服务开发
  
- **模块划分**：用户管理、维修人员管理、工单管理、库存管理、财务管理API
- **核心功能**：
- 工单自动分配算法（按工种/负载分配）
- 事务控制（批量工单提交、月度结算事务）
- 数据统计接口（车型维修统计、工种任务占比等多表连接查询）
- 安全机制（JWT登录认证、权限控制，管理员/用户/维修人员角色区分）
  
  #### 3. 技术实现
  
- 使用Spring Boot Web开发RESTful API
- 集成MyBatis-Plus简化数据库操作
- 实现Swagger接口文档，方便前端调用
  
  ### **成员B：前端开发（Vue）**
  
  #### 1. 界面设计与交互
  
- 实现用户端页面：注册登录、报修提交、记录查询、反馈评分
- 实现维修人员端页面：工单处理、材料记录、收入查询
- 实现管理员端页面：数据管理、统计监控
- 响应式设计（适配PC/移动端），使用Element-UI组件库提升开发效率
  
  #### 2. 功能开发
  
- 调用后端API完成数据交互（Axios封装）
- 状态管理：使用Vuex存储用户登录状态、工单列表等全局数据
- 表单验证（报修信息提交、用户信息修改）
- 可视化统计：使用ECharts展示维修次数、成本比例等图表
  
  #### 3. 联调与测试
  
- 对接后端API，验证数据一致性（如工单状态更新、费用计算）
- 用户体验优化（加载动画、错误提示、操作反馈）
  
  ### **协作重点**
  

1. **数据库设计阶段**：共同确认ER图和表结构，确保业务逻辑完整（如工单与维修人员多对多关系）。
2. **接口定义阶段**：制定API文档（URL、请求方法、参数格式），明确前后端交互规则（如工单分配返回状态码）。
3. **联调阶段**：

- 后端完成API后，前端使用Postman测试接口可用性；
- 共同验证复杂逻辑（如工单删除级联操作、事务回滚机制）。

4. **文档与部署**：

- 后端负责数据库初始化脚本、运行说明文档；
- 前端打包可执行程序包（npm run build），配合后端完成Docker部署（可选）。
  
  ### **时间分配建议**
  
  | 阶段  | 任务  | 时间占比 |
  | --- | --- | --- |
  | 需求分析 | 共同梳理功能与分工 | 10% |
  | 数据库设计 | 共同设计ER图与表结构 | 15% |
  | 后端开发 | 接口实现、事务处理、存储过程 | 35% |
  | 前端开发 | 页面实现、交互设计、API调用 | 30% |
  | 联调与测试 | 功能验证、模拟数据测试 | 10% |
  
  ### **风险与应对**
  
- **技术难点**：工单自动分配算法、多表复杂统计查询。
- 解决方案：查阅类似调度算法（如轮询法、负载均衡），使用SQL窗口函数或Spring框架辅助统计。
